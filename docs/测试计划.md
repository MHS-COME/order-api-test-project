## 2. 测试策略

### 2.1 接口测试优先级矩阵

```
            影响高
              │
    P1 ★★★    │    P0 ★★★★
   (重要但不紧急) │   (紧急且重要)
              │
  ────────────┼──────────── 影响低
              │
    P3 ★      │    P2 ★★
   (低优先级)  │   (一般)
              │
           紧急低
```

| 优先级 | 定义 | 覆盖内容 | 占比 |
|--------|------|---------|------|
| **P0** | 核心业务流程，阻塞即不可发布 | 注册 → 登录 → 创建订单 → 支付 → 查询 全链路 + UI E2E | 20% |
| **P1** | 高频接口的关键异常路径 | 必填参数缺失、未授权访问、凭证错误、支付金额校验 | 35% |
| **P2** | 一般异常场景 | 越权访问、资源不存在、状态冲突、支付幂等性 | 30% |
| **P3** | 边界值与安全专项 | 参数边界值、JWT 攻击、XSS/SQL 注入、Token 过期 | 15% |

### 2.2 自动化覆盖目标

| 优先级 | 自动化目标 | 说明 |
|--------|-----------|------|
| P0 | **100%** | 每次提交必须通过，作为 CI 门禁 |
| P1 | **95%** | 每日构建自动执行 |
| P2 | **85%** | 每轮回归自动执行 |
| P3 | **70%** | 发版前手动触发 |

### 2.3 测试执行节奏

```
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│ 冒烟测试  │ →  │ 功能测试  │ →  │ 回归测试  │ →  │ 发布验证  │
│ (5 min)  │    │ (15 min) │    │ (25 min) │    │ (15 min) │
└──────────┘    └──────────┘    └──────────┘    └──────────┘
  每次提交        每次 MR         每日构建         发版前
  6 模块并行       P0+P1 全量      P0~P3 全量       P0 全量 + 性能 + UI
```

## 4. 风险与应对

| 编号 | 风险项 | 概率 | 影响 | 等级 | 应对措施 |
|------|--------|------|------|------|---------|
| R1 | **测试环境不稳定**（Mock 服务宕机、网络中断导致用例大量误报） | 中 | 高 | 🔴 高 | ① 执行前自动健康检查（`GET /health`）<br>② CI 中自动启动 Mock 服务<br>③ 失败用例自动重试 1 次 |
| R2 | **被测接口变更未同步**（字段改名、路径调整导致用例大量失败） | 高 | 中 | 🟡 中 | ① 要求接口变更时同步更新 Collection<br>② 用例与 API 文档保持同步 |
| R3 | **测试数据污染**（并发执行时数据相互干扰） | 中 | 中 | 🟡 中 | ① 每次执行前 `POST /__reset` 重置数据<br>② 使用动态生成唯一数据<br>③ 执行后自动清理 |
| R4 | **Token 过期导致的测试中断**（长链路执行超过 Token 有效期） | 中 | 低 | 🟢 低 | ① 各模块 pre-request 自动检查 Token<br>② 过期前调用 `/refresh` 刷新<br>③ CI 中全局变量传递 token |
| R5 | **Selenium 浏览器兼容性**（Chrome 版本与 ChromeDriver 不匹配） | 中 | 中 | 🟡 中 | ① webdriver-manager 自动匹配版本<br>② CI 支持无头模式（--headless）<br>③ CI 缓存 ChromeDriver |
| R6 | **k6 资源限制**（本地压测占用过高资源影响其他任务） | 低 | 低 | 🟢 低 | ① 默认 20 VU 轻量压测<br>② 大并发压测使用独立环境<br>③ 压测前检查系统资源 |
| R7 | **GitHub Actions 制品过期**（报告留存超过 30 天被删除） | 低 | 低 | 🟢 低 | ① 重要报告下载到本地存档<br>② Allure 报告本地生成并保留<br>③ 关键指标记录到 README |

## 6. 附录

### 6.1 术语表

| 术语 | 说明 |
|------|------|
| **SUT** | System Under Test，被测系统 |
| **E2E** | End-to-End，端到端全链路测试 |
| **P0-P3** | 优先级分级（Priority 0-3） |
| **VU** | Virtual User，虚拟并发用户（k6） |
| **P95** | 95 分位响应时间，即 95% 的请求在此时间内完成 |
| **DDT** | Data-Driven Testing，数据驱动测试 |
| **JWT** | JSON Web Token，无状态认证令牌 |
| **SPA** | Single Page Application，单页应用 |
