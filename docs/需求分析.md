> 版本：v1.0  
> 更新时间：2026-06-03

---

## 1. 项目概述

**OrderAPITest** 是一个企业级 API 自动化测试项目，针对典型的订单系统核心接口进行全面的功能验证与异常覆盖。测试范围覆盖用户认证、订单生命周期管理（创建 → 查询 → 取消）的完整链路。

### 1.1 测试目标

| 目标 | 说明 |
|------|------|
| 功能正确性 | 验证每个接口在合法输入下返回预期结果 |
| 异常容错性 | 验证参数缺失、未授权、资源不存在等异常场景的响应 |
| 数据一致性 | 验证订单状态变更在整个生命周期中的一致性 |
| 安全性 | 验证未授权访问被正确拦截 |

### 1.2 被测系统假设

- 协议：HTTP/HTTPS
- 数据格式：JSON
- 认证方式：Bearer Token（JWT）
- 基础 URL（测试环境）：`http://api.example.test/v1`

## 2. 接口详细规格

### 2.1 用户登录 — `POST /login`

#### 请求规格

| 项目 | 内容 |
|------|------|
| URL | `/login` |
| 方法 | `POST` |
| Content-Type | `application/json` |
| 认证 | 无需 |

**请求体参数：**

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| `username` | string | 是 | 用户名，4-32 字符 |
| `password` | string | 是 | 密码，6-64 字符 |

#### 预期成功响应

**HTTP Status: `200 OK`**

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIs...",
    "tokenType": "Bearer",
    "expiresIn": 7200,
    "userId": 10001,
    "username": "testuser"
  }
}
```

**响应字段说明：**

| 字段 | 类型 | 说明 |
|------|------|------|
| `code` | int | 业务状态码，0 表示成功 |
| `message` | string | 提示信息 |
| `data.token` | string | JWT 访问令牌 |
| `data.tokenType` | string | 令牌类型，固定为 "Bearer" |
| `data.expiresIn` | int | 令牌有效期（秒） |
| `data.userId` | int | 用户唯一标识 |
| `data.username` | string | 用户名 |

#### 异常场景

| 场景编号 | 场景描述 | 预期 HTTP 状态 | 预期响应体（核心字段） |
|----------|---------|---------------|---------------------|
| LOGIN-ERR-01 | 缺少必填字段 `username` | `400 Bad Request` | `{"code": 1001, "message": "参数校验失败：username 不能为空"}` |
| LOGIN-ERR-02 | 缺少必填字段 `password` | `400 Bad Request` | `{"code": 1001, "message": "参数校验失败：password 不能为空"}` |
| LOGIN-ERR-03 | 用户名或密码错误 | `401 Unauthorized` | `{"code": 1002, "message": "用户名或密码错误"}` |
| LOGIN-ERR-04 | 账户被锁定（连续 5 次失败） | `423 Locked` | `{"code": 1003, "message": "账户已被锁定，请 15 分钟后重试"}` |

### 2.2 创建订单 — `POST /orders`

#### 请求规格

| 项目 | 内容 |
|------|------|
| URL | `/orders` |
| 方法 | `POST` |
| Content-Type | `application/json` |
| 认证 | Bearer Token（Header: `Authorization: Bearer <token>`） |

**请求头：**

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| `Authorization` | string | 是 | `Bearer <token>`，由登录接口获取 |

**请求体参数：**

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| `productId` | int | 是 | 商品 ID，正整数 |
| `quantity` | int | 是 | 购买数量，1-999 |
| `addressId` | int | 是 | 收货地址 ID，正整数 |
| `remark` | string | 否 | 订单备注，最长 200 字符 |

#### 预期成功响应

**HTTP Status: `201 Created`**

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "orderId": 202606030001,
    "orderNo": "ORD202606030001",
    "status": "PENDING",
    "totalAmount": 299.00,
    "productId": 5001,
    "productName": "无线蓝牙耳机",
    "quantity": 2,
    "createdAt": "2026-06-03T10:30:00+08:00"
  }
}
```

**响应字段说明：**

| 字段 | 类型 | 说明 |
|------|------|------|
| `data.orderId` | int | 订单内部 ID |
| `data.orderNo` | string | 订单编号（对外展示） |
| `data.status` | string | 订单状态：PENDING（待处理） |
| `data.totalAmount` | float | 订单总金额 |
| `data.productId` | int | 商品 ID（回显） |
| `data.productName` | string | 商品名称 |
| `data.quantity` | int | 购买数量（回显） |
| `data.createdAt` | string | 创建时间（ISO 8601） |

#### 异常场景

| 场景编号 | 场景描述 | 预期 HTTP 状态 | 预期响应体（核心字段） |
|----------|---------|---------------|---------------------|
| ORDER-ERR-01 | 未携带 Authorization 头 | `401 Unauthorized` | `{"code": 2001, "message": "未提供认证令牌"}` |
| ORDER-ERR-02 | Token 已过期 | `401 Unauthorized` | `{"code": 2002, "message": "令牌已过期，请重新登录"}` |
| ORDER-ERR-03 | 缺少必填字段 `productId` | `400 Bad Request` | `{"code": 1001, "message": "参数校验失败：productId 不能为空"}` |
| ORDER-ERR-04 | 缺少必填字段 `quantity` | `400 Bad Request` | `{"code": 1001, "message": "参数校验失败：quantity 不能为空"}` |
| ORDER-ERR-05 | 缺少必填字段 `addressId` | `400 Bad Request` | `{"code": 1001, "message": "参数校验失败：addressId 不能为空"}` |
| ORDER-ERR-06 | `productId` 对应的商品不存在 | `404 Not Found` | `{"code": 2003, "message": "商品不存在或已下架"}` |
| ORDER-ERR-07 | `quantity` 超出库存 | `400 Bad Request` | `{"code": 2004, "message": "库存不足，当前剩余 5 件"}` |

### 2.3 查询订单详情 — `GET /orders/{id}`

#### 请求规格

| 项目 | 内容 |
|------|------|
| URL | `/orders/{id}` |
| 方法 | `GET` |
| Content-Type | 无请求体 |
| 认证 | Bearer Token |

**路径参数：**

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| `id` | int | 是 | 订单 ID（`orderId`），正整数 |

**请求头：**

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| `Authorization` | string | 是 | `Bearer <token>` |

#### 预期成功响应

**HTTP Status: `200 OK`**

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "orderId": 202606030001,
    "orderNo": "ORD202606030001",
    "status": "PENDING",
    "totalAmount": 299.00,
    "items": [
      {
        "productId": 5001,
        "productName": "无线蓝牙耳机",
        "price": 149.50,
        "quantity": 2,
        "subtotal": 299.00
      }
    ],
    "address": {
      "recipient": "张三",
      "phone": "138****5678",
      "province": "广东省",
      "city": "深圳市",
      "detail": "南山区科技园路 1 号"
    },
    "createdAt": "2026-06-03T10:30:00+08:00",
    "updatedAt": "2026-06-03T10:30:00+08:00"
  }
}
```

**响应字段说明：**

| 字段 | 类型 | 说明 |
|------|------|------|
| `data.items` | array | 订单商品明细列表 |
| `data.items[].price` | float | 商品单价 |
| `data.items[].subtotal` | float | 小计金额 |
| `data.address` | object | 收货地址信息 |
| `data.updatedAt` | string | 最后更新时间（ISO 8601） |

#### 异常场景

| 场景编号 | 场景描述 | 预期 HTTP 状态 | 预期响应体（核心字段） |
|----------|---------|---------------|---------------------|
| GET-ERR-01 | 订单 ID 不存在 | `404 Not Found` | `{"code": 3001, "message": "订单不存在"}` |
| GET-ERR-02 | 查询非本人订单（越权） | `403 Forbidden` | `{"code": 3002, "message": "无权访问该订单"}` |
| GET-ERR-03 | 订单 ID 格式非法（非数字） | `400 Bad Request` | `{"code": 1001, "message": "参数校验失败：id 必须为正整数"}` |
| GET-ERR-04 | 未携带 Authorization 头 | `401 Unauthorized` | `{"code": 2001, "message": "未提供认证令牌"}` |

### 2.4 取消订单 — `PUT /orders/{id}/cancel`

#### 请求规格

| 项目 | 内容 |
|------|------|
| URL | `/orders/{id}/cancel` |
| 方法 | `PUT` |
| Content-Type | `application/json` |
| 认证 | Bearer Token |

**路径参数：**

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| `id` | int | 是 | 订单 ID（`orderId`），正整数 |

**请求头：**

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| `Authorization` | string | 是 | `Bearer <token>` |

**请求体参数：**

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| `reason` | string | 否 | 取消原因，最长 200 字符 |

#### 预期成功响应

**HTTP Status: `200 OK`**

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "orderId": 202606030001,
    "orderNo": "ORD202606030001",
    "status": "CANCELLED",
    "totalAmount": 299.00,
    "cancelledAt": "2026-06-03T11:00:00+08:00"
  }
}
```

**响应字段说明：**

| 字段 | 类型 | 说明 |
|------|------|------|
| `data.status` | string | 订单状态变为 `CANCELLED` |
| `data.cancelledAt` | string | 取消时间（ISO 8601） |

#### 异常场景

| 场景编号 | 场景描述 | 预期 HTTP 状态 | 预期响应体（核心字段） |
|----------|---------|---------------|---------------------|
| CANCEL-ERR-01 | 订单 ID 不存在 | `404 Not Found` | `{"code": 3001, "message": "订单不存在"}` |
| CANCEL-ERR-02 | 取消非本人订单（越权） | `403 Forbidden` | `{"code": 3002, "message": "无权操作该订单"}` |
| CANCEL-ERR-03 | 订单状态不允许取消（如已发货） | `409 Conflict` | `{"code": 4001, "message": "当前订单状态不允许取消", "data": {"currentStatus": "SHIPPED"}}` |
| CANCEL-ERR-04 | 重复取消同一订单（幂等性） | `409 Conflict` | `{"code": 4002, "message": "订单已被取消，无法重复操作"}` |
| CANCEL-ERR-05 | 未携带 Authorization 头 | `401 Unauthorized` | `{"code": 2001, "message": "未提供认证令牌"}` |

## 3. 业务状态码汇总

| 业务码 | HTTP 状态 | 含义 |
|--------|----------|------|
| `0` | 200/201 | 操作成功 |
| `1001` | 400 | 参数校验失败 |
| `1002` | 401 | 用户名或密码错误 |
| `1003` | 423 | 账户已锁定 |
| `2001` | 401 | 未提供认证令牌 |
| `2002` | 401 | 令牌已过期 |
| `2003` | 404 | 商品不存在或已下架 |
| `2004` | 400 | 库存不足 |
| `3001` | 404 | 订单不存在 |
| `3002` | 403 | 无权访问/操作该订单 |
| `4001` | 409 | 订单状态不允许取消 |
| `4002` | 409 | 订单已被取消（重复操作） |

## 4. 测试策略

### 4.1 测试分层

```
┌─────────────────────────────────┐
│        E2E 场景测试（全链路）      │  ← 登录 → 创建订单 → 查询 → 取消
├─────────────────────────────────┤
│        接口级功能测试             │  ← 单接口正向 + 异常覆盖
├─────────────────────────────────┤
│        契约测试（Schema）         │  ← 响应结构、字段类型、必填字段校验
└─────────────────────────────────┘
```

### 4.2 测试优先级

| 优先级 | 覆盖内容 |
|--------|---------|
| **P0** | 正向流程：登录 → 创建订单 → 查询 → 取消（全链路） |
| **P1** | 各接口必填参数缺失校验、未授权访问拦截 |
| **P2** | 越权访问、资源不存在、状态冲突 |
| **P3** | 边界值（quantity 上下限）、幂等性、Token 过期 |

### 4.3 建议技术栈

| 组件 | 选型建议 |
|------|---------|
| 编程语言 | Python 3.10+ / Java 17+ |
| 测试框架 | pytest（Python）或 TestNG + RestAssured（Java） |
| HTTP 客户端 | requests（Python）或 OkHttp（Java） |
| 断言库 | pytest 内置 assert / Hamcrest / AssertJ |
| 测试报告 | Allure Report |
| CI 集成 | Jenkins / GitHub Actions |
| 数据驱动 | JSON / YAML / CSV 测试数据文件 |
